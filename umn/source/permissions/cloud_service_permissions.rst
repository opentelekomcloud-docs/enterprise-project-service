Cloud Service Permissions
=========================

IAM allows you to control resource access by enterprise project. You can assign users or user groups different permissions for them to access resources in different enterprise projects. For services supported by Enterprise Project Management, see :ref:`Supported Services <em_01_0005>`. For default permissions of each service, see `Permissions <https://docs.otc.t-systems.com/permissions/index.html>`__

When the services supported by EPS are associated with services that are not, additional permissions are demanded to manage them. You can use the following methods to configure dependent policies:

-  On the IAM console, add policies to user groups. For details, see `Creating a User Group and Assigning Permissions <https://docs.otc.t-systems.com/usermanual/iam/iam_01_0030.html>`__.
-  Create custom policies on the IAM console. For details, see `Creating a Custom Policy <https://docs.otc.t-systems.com/identity-access-management/umn/user_guide/permissions/creating_a_custom_policy.html>`__.
